# 技能石列表与手机端优化迁移说明

本文档用于在测试服助手脚本更新后，重新补回本次围绕技能石列表和手机竖屏体验追加的需求。

当前参考文件：`测试服助手2.162.txt`

## 目标功能

本组需求主要覆盖技能石页：

- 仅按移动手机端竖屏优化，不追求横屏布局。
- 技能石多选列表不能因为文本太长导致关键信息不可见。
- 技能石列表长文本优先换行展示，而不是压缩成更短文本。
- 装备上的技能石来源优先显示装备槽位，例如 `[主手]`、`[副手]`，而不是长装备名。
- 装备槽位英文 key 要映射为中文，例如 `glove`、`boot`、`necklace`、`ring`。
- 按钮区域减少浪费：主操作按钮双列，次要按钮小胶囊化，停止按钮单独高亮。
- 手机端点击筛选/输入时，底部键盘不应完全遮住操作区域。

迁移时要确认：

- `mainHand` 显示为 `主手`。
- `offHand` 显示为 `副手`。

- `boot/boots` 不再显示英文，显示 `鞋子`。
- `necklace/amulet` 不再显示英文，显示 `项链`。
- `ring/ring1/ring2` 不再显示英文，显示 `戒指/左戒/右戒`。

## slotLabel 传递链路

装备技能石列表必须一路保留 `slotLabel`。

关键链路：

1. `getCharacterEquipmentEntries` 从角色装备槽位生成 `slotLabel`。
2. `fetchEquippedSkillStones` 写入每颗装备技能石的 `slotLabel`。
3. `enrichSkillStoneWithDetail` 调用时继续传入 `slotLabel`。
4. `normalizeSkillStone` 保留 `slotLabel` 字段。
5. `formatSkillStoneLabel` 对装备技能石优先用 `slotLabel` 展示。

`formatSkillStoneLabel` 关键逻辑：

```js
const formatSkillStoneLabel = (stone) => {
  const corruptedText = stone.corrupted ? ' 已腐化' : '';
  const sourceText = stone.source === 'equipment'
    ? (stone.slotLabel || stone.sourceLabel || '装备')
    : (stone.sourceLabel || '背包');
  return `[${sourceText}] ${stone.name} | Lv.${stone.level || '?'} | 品质 ${stone.quality || 0}%${corruptedText}`;
};
```

注意：

- 列表可见文本用 `[主手]`、`[副手]` 这类槽位名。
- `equipmentName` 可以保留在 `title` 中，方便长按/悬停查看完整装备名。
- 不要把列表可见文本恢复成 `[装备：导师的工艺之...短杖]`，会在手机端挤掉技能石等级和品质信息。

## 技能石列表渲染

`renderSkillStoneOptions` 中每个 option 要使用 `formatSkillStoneLabel(stone)`。

参考结构：

```js
const fullLabel = `[${stone.sourceLabel}] ${stone.name} | Lv.${stone.level || '?'} | 品质 ${stone.quality || 0}%${stone.corrupted ? ' 已腐化' : ''}`;
const optionElement = createElement('option', {
  value: stone.id,
  textContent: formatSkillStoneLabel(stone),
});
optionElement.title = `${fullLabel}${stone.equipmentName ? ` | ${stone.equipmentName}` : ''}`;
```

迁移时要确认：

- 背包技能石显示 `[背包] 技能名 | Lv.x | 品质 x%`。
- 装备技能石显示 `[主手] 技能名 | Lv.x | 品质 x%`。
- 腐化技能石追加 `已腐化`。
- 长装备名只进入 `title`，不挤占列表可见文本。

## 长文本可读性

历史问题：

`[背包] 快速攻击(辅) | Lv.20 | 品质 20% 已腐化`

这类文本在手机端容易一行溢出，看不到后面的等级、品质、腐化状态。

需求结论：

- 不要压缩成更短文案。
- 不要依赖横向滚动作为唯一方案。
- 可视区域需要能看到完整信息。
- 一行显示不全时允许换行。

当前脚本是原生 `select multiple`，不同浏览器对 `option` 换行支持不完全一致。迁移时至少要保留以下 CSS 意图：

```css
.poe2-stone-select {
  height: 240px !important;
}

@media (max-width: 640px) {
  select.poe2-input[multiple],
  .poe2-stone-select {
    height: 190px !important;
  }
}
```

如果新版浏览器/脚本中原生 `select option` 仍无法换行，需要升级为自定义多选列表：

- 用 `div[role="listbox"]` 替代原生多选 select。
- 每个技能石用可点击行或 checkbox 行展示。
- 行样式使用 `white-space: normal; overflow-wrap: anywhere; word-break: break-word;`。
- 保留多选语义和当前选中 ID 读取逻辑。

## 手机竖屏布局

移动端样式锚点：`@media (max-width: 640px)`。

核心要求：

- 面板宽度贴合手机屏幕：`width: calc(100vw - 16px)`。
- 面板高度使用 `100dvh`，减少键盘弹出后的遮挡。
- 顶部 tab/subtab 横向滚动。
- 输入框高度适配触控，字体至少 `16px`，避免 iOS 自动放大。
- 技能石多选列表高度降低，给按钮和日志留空间。

现有关键 CSS：

```css
@media (max-width: 640px) {
  .poe2-panel {
    left: 8px !important;
    right: auto;
    top: 8px !important;
    width: calc(100vw - 16px);
    max-height: calc(100dvh - 16px);
  }

  .poe2-input {
    height: 36px;
    font-size: 16px;
  }

  select.poe2-input[multiple],
  .poe2-stone-select {
    height: 190px !important;
  }
}
```

## 按钮区域

历史需求：

- 按钮区域太大。
- 一个按钮占一行浪费空间。
- 主操作按钮双列。
- 次要按钮小胶囊化。
- 停止按钮单独高亮。

现有移动端基础：

```css
@media (max-width: 640px) {
  .poe2-actions {
    display: grid;
    grid-template-columns: 1fr 1fr;
    gap: 8px;
    width: 100%;
  }

  .poe2-actions .poe2-button {
    width: 100%;
    min-height: 36px;
    padding: 0 8px;
    white-space: normal;
    line-height: 1.2;
  }
}
```

技能石区按钮锚点：`createSkillStoneSection`。

按钮包括：

- `加载技能石`
- `全选`
- `全选(除清晰精准活力)`
- `清空选择`
- `升级选中技能石`
- `使用宝石匠的棱镜`
- `智能练技能`
- `使用瓦尔宝珠`
- `丢弃选中技能石`
- 停止按钮

迁移时要保持：

- `加载技能石` 为主按钮之一。
- 升级、棱镜、智能练技能为主操作。
- 丢弃/瓦尔视觉上有警告色。
- 停止按钮使用 `poe2-stop` 或专用停止样式，不能混成普通按钮。
- 在窄屏下按钮至少双列，不要全部单列铺满。

## 防止键盘遮挡

历史问题：

点击筛选技能石时，底部弹出输入键盘后，页面会被按钮区域挡住。

迁移时检查：

- 面板使用 `max-height: calc(100dvh - 16px)`，不是固定 `100vh`。
- 面板内部 `overflow: auto`。
- 技能石列表高度在移动端缩短。
- 按钮区不要固定在底部覆盖内容，除非额外加底部 padding。
- 如果新增底部固定操作栏，需要加 `padding-bottom: calc(操作栏高度 + env(safe-area-inset-bottom))`。

## 验证清单

迁移后建议检查以下场景：

- 手机竖屏打开助手，切到 `技能石` 页，面板不超出屏幕。
- 点击输入框或筛选类输入时，键盘弹出后仍能滚动到按钮和列表。
- 加载技能石后，背包项显示 `[背包]`。
- 主手装备技能石显示 `[主手]`，副手装备技能石显示 `[副手]`。
- 手套、鞋子、项链、戒指位置不显示英文 key。
- 长文本技能石能看到等级、品质、腐化状态。
- 选中多个技能石后，升级/棱镜/腐化/丢弃只作用于选中项。
- 停止按钮仍可见，且样式明显区别于普通按钮。
