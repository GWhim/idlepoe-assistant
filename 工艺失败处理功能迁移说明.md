# 工艺失败处理功能迁移说明

本文记录本次在 `测试服助手2.17` 上新增的“步骤动作 -> 动作类型 -> 工艺/智能工艺失败处理”功能，方便后续原作者版本更新后重新迁移。

## 功能目标

- 连续打造步骤中，`工艺` 和 `智能工艺` 不再只有“完成后”分支。
- 当工艺接口被后端拒绝时，步骤返回失败，并走与“条件判断 -> 条件不成立”一致的失败处理选项。
- 失败处理支持原有选项：跳转到步骤、重铸后从步骤 A 开始、终止(打造成功)、终止(异常错误)、终止(手动操作)。
- 普通配置错误仍然抛错，例如未选择工艺、装备类型不匹配、腐化装备不允许使用该工艺。

## 迁移清单

1. 修改 `applyCraftBench`

- 给 `applyCraftBench(equipment, craftId, options)` 增加或保留 `continueOnBackendRejection` 参数。
- 当后端请求失败且 `continueOnBackendRejection === true` 时，不抛错，记录警告日志并返回 `false`。
- 日志建议使用通用文案：`工艺请求被后端拒绝`，避免误导为一定是多大师导致。

2. 修改 `applySmartCraftBench`

- 给 `applySmartCraftBench` 增加 `routeBackendRejection` 参数。
- 调用 `applyCraftBench` 时传入 `continueOnBackendRejection: routeBackendRejection || hasMultiMasterCraft`。
- 当 `routeBackendRejection` 为 true 且实际未成功时，返回 `false`，让外层连续打造进入失败处理。

3. 修改 `executeContinuousCraftStep`

- `smartCraftBench` 调用 `applySmartCraftBench(equipment, craftId, { routeBackendRejection: true })`。
- `craftBench` 调用 `applyCraftBench(equipment, craftId, { continueOnBackendRejection: true })`。
- `craftBench` 根据返回值记录“工艺完成”或“工艺失败，进入失败处理”，并把布尔值返回给路由层。

4. 增加失败分支动作判断

建议新增两个小工具函数：

```js
const isContinuousFailureRoutableAction = (action) => (
  action === 'conditionCheck' || action === 'craftBench' || action === 'smartCraftBench'
);

const getContinuousFailureBranchLabel = (action) => (
  action === 'conditionCheck' ? '条件不成立' : '失败'
);
```

5. 修改连续打造路由

- `handleContinuousStepRouting` 中，失败结果如果属于 `conditionCheck`、`craftBench`、`smartCraftBench`，使用 `failureHandling` 和 `failureTargetStepIndex`。
- 其他动作失败仍按原有逻辑处理，不额外展示失败分支。
- 失败跳转默认 fallback 仍是当前步骤，防止未选目标时直接跳下一步。

6. 修改流程分析和导出校验

- `getContinuousRoutingOutcomesForAnalysis` 要把 `craftBench`、`smartCraftBench` 的失败分支纳入可达性分析。
- `assertContinuousCraftFlowHasExit` 要检查工艺失败分支不能跳回自身，规则与条件不成立一致。
- `getContinuousStepFutureTargetLabels` 要把工艺失败跳转纳入“步骤（新增后）”检测。

7. 修改 UI 显示

- `updateContinuousHandlingTargetVisibility` 中，失败处理栏对 `conditionCheck`、`craftBench`、`smartCraftBench` 显示。
- 条件判断时失败栏标签显示 `条件不成立` / `不成立跳转步骤`。
- 工艺和智能工艺时失败栏标签显示 `失败后` / `失败跳转步骤`。
- 步骤摘要中，工艺类步骤同时显示“完成后”和“失败”路径。

8. 修改帮助文案

- 将“这个设置只对条件判断步骤生效”改为“这个设置对条件判断、工艺、智能工艺步骤生效”。
- 说明：条件判断失败表示条件不成立；工艺失败表示工艺请求被后端拒绝。

## 验证方式

```powershell
node -e "const fs=require('fs'); const vm=require('vm'); const text=fs.readFileSync('测试服助手2.17.txt','utf8'); new vm.Script(text); console.log('ok')"
```

同时确认：

- `idlepoe-assistant.user.js` 第一行必须是 `// ==UserScript==`。
- `@version` 与当前原作者版本一致。
- 不要重新带入已取消的本地方案，例如单手双加一法杖、全属性占位、长杖法术技能石加五。
