## 默认方案注入

需要新增内置方案 ID：

```js
const BUILTIN_CRAFT_PLAN_IDS = {
  longStaffDualPlusOneAltAugRegal: 'builtin-long-staff-dual-plus-one-alt-aug-regal',
};
```

本地方案合并逻辑需要覆盖同 ID 内置方案，确保脚本更新后能刷新旧本地方案：

```js
const mergeBuiltinCraftPlans = (rawPlans) => {
  const normalizedPlans = normalizeCraftPlanList(rawPlans);
  const planMap = new Map(normalizedPlans.map((plan) => [plan.id, plan]));
  getBuiltinCraftPlans().forEach((plan) => {
    if (!plan) return;
    planMap.set(plan.id, plan);
  });
  return [...planMap.values()].sort((left, right) => right.updatedAt - left.updatedAt);
};
```

`persistCraftPlans` 中使用：

```js
state.craftPlans = mergeBuiltinCraftPlans(state.craftPlans);
```

const createTargetConditionGroups = (targetConditions, minRequired) => [normalizeAffixConditionGroup({

## 工艺解析

新增内置工艺 token：

```js
const BUILTIN_CONTINUOUS_CRAFT_TOKENS = {
  intelligence15To20Suffix: '__builtin_intelligence_15_to_20_suffix__',
};
```

运行时从工艺列表查找：

- 位置：后缀
- 装备类型：当前装备可用
- 文本包含：`智慧`
- 数值包含：`15` 和 `20`

```js
const findIntelligence15To20SuffixCraft = (categoryValue, equipment) => {
  const equipmentMask = equipment ? parseEquipmentTypeMask(equipment?.equipmentType) : getCraftBenchMaskByCategory(categoryValue);
  return (Array.isArray(state.craftBench?.list) ? state.craftBench.list : [])
    .filter((craft) => getCraftBenchPositionType(craft) === 'suffix')
    .filter((craft) => isCraftBenchForMask(craft, equipmentMask))
    .filter((craft) => /智慧/.test(String(craft?.searchText || craft?.label || '')))
    .find((craft) => {
      const numbers = (String(craft?.searchText || craft?.label || '').match(/\d+/g) || [])
        .map((value) => Number.parseInt(value, 10))
        .filter(Number.isFinite);
      return numbers.includes(15) && numbers.includes(20);
    }) || null;
};
```

步骤列表展示兜底文字：

```js
const builtinCraftText = getContinuousBuiltinCraftToken(step) === BUILTIN_CONTINUOUS_CRAFT_TOKENS.intelligence15To20Suffix
  ? '后缀：+15 ~ 20 智慧'
  : '';
```

## 状态机步骤

默认方案步骤如下，字母为 UI 中显示的步骤顺序：

| 步骤 | 动作 | 成立/完成 | 不成立 |
| --- | --- | --- | --- |
| A | 变为魔法 | B | - |
| B | 判断 1/3 | D | C |
| C | 改造石 | D | - |
| D | 智能增幅 | E | - |
| E | 判断 1/3 | F | A |
| F | 富豪石 | G | - |
| G | 判断 2/3 | 成功 | H |
| H | 判断 1/3 | I | A |
| I | 智能工艺：后缀 +15~20 智慧 | J | - |
| J | 判断 1/3 | K | A |
| K | 判断 2/3 | 成功 | L |
| L | 判断前缀数 = 3 | Q | M |
| M | 崇高石 | N | - |
| N | 判断 2/3 | 成功 | O |
| O | 判断 1/3 | P | A |
| P | 判断保留前缀：锋芒的 | L | Q |
| Q | 剥离石 | R | - |
| R | 判断 2/3 | 成功 | S |
| S | 判断 1/3 | T | A |
| T | 判断是否有工艺词缀 | L | I |

关键行为：

- 使用改造石前先做 `1/3` 判断，已有目标词缀时跳过改造，避免洗掉。
- 富豪后只要 `1/3` 还在，就进入工艺崇高剥离循环。
- 前缀满 `3/3` 时直接剥离，避免在保留前缀判断和前缀满判断之间空转。
- 崇高后如果出 `锋芒的` 且前缀未满，回到前缀满判断，继续崇高。
- 崇高后如果是垃圾结果，立即剥离。
- 剥离后如果 `1/3` 没了，从 A 重做。
- 剥离后如果 `1/3` 还在且工艺还在，回到 L 判断前缀是否满。
- 剥离后如果 `1/3` 还在但工艺没了，回到 I 重新上工艺。

## 剥离/崇高日志

日志等级为“主要”时，只在自定义打造使用 `剥离石`、`崇高石` 后打印当前全部词缀。

新增格式化函数：

```js
const formatEquipmentAffixSnapshot = (equipment) => {
  const affixes = Array.isArray(equipment?.affixes) ? equipment.affixes : [];
  return affixes.length ? affixes.map((affix) => {
    const positionText = formatAffixPositionName(affix?.type);
    return `${positionText} ${formatCraftSnapshotAffix(affix)}`;
  }).join('；') : '无';
};
```

执行循环中触发条件：

```js
if (stepMatched && ['annulment', 'exalted'].includes(currentStep.action)) {
  logContinuousCraftAffixSnapshot(
    equipment,
    stepIndex,
    CONTINUOUS_CRAFT_ACTIONS[currentStep.action]?.label || currentStep.action,
  );
}
```
